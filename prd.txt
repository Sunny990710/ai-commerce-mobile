# PRD: 대화형 패션 커머스 앱 `StyleChat` (가칭)

문서 버전: v1.0  
문서 목적: 웹 프로토타입을 기반으로 iOS/Android 앱 개발이 가능한 수준의 제품 요구사항 정의  
작성 기준: 현재 웹 프로토타입의 핵심 흐름인 `대화 검색 -> 상품 추천 -> 상품 상세 -> 가격 비교/추적 -> 저장/위시 -> AI 옷장 -> 결제`를 모바일 앱 중심으로 재구성

---

## 1. 제품 개요

### 1-1. 제품 비전
사용자와 자연어로 대화하며 취향, 체형, 예산, TPO(시간/장소/상황)에 맞는 패션 상품을 추천하고 구매까지 연결해 주는 `내 손안의 AI 퍼스널 스타일리스트 앱`을 만든다.

### 1-2. 제품 한 줄 설명
스크롤 기반 쇼핑몰이 아니라 `대화 중심 탐색`을 메인 인터페이스로 삼는 패션 커머스 앱.

### 1-3. 해결하려는 문제
기존 패션 커머스는 다음 문제를 가진다.

1. 사용자는 원하는 스타일을 명확한 키워드로 검색하기 어렵다.
2. 상품 수가 많아 비교 피로가 크고, 코디 판단이 어렵다.
3. 상품 상세 정보, 리뷰, 혜택, 가격 비교가 분산돼 구매 결정을 늦춘다.
4. 이미 가진 옷과 어울리는지 확인하기 어렵다.
5. 가격이 떨어질지, 더 좋은 대안이 있는지 판단하기 어렵다.

### 1-4. 핵심 가치 제안
1. 검색 대신 대화로 원하는 룩을 빠르게 찾는다.
2. AI가 이유 있는 추천을 제공해 탐색 시간을 줄인다.
3. 상품 상세, 가격 비교, 리뷰 요약, 위시, 구매까지 한 흐름에서 끝낸다.
4. 내 옷장과 연결해 실제 활용도를 높인다.
5. 가격 추적과 재입고 알림으로 구매 타이밍까지 지원한다.

### 1-5. 차별점
1. 홈 자체가 상품 리스트가 아니라 대화형 스타일 탐색 화면이다.
2. 추천 결과가 단순 검색 결과가 아니라 `추천 이유 + 상품 카드 + 다음 질문`으로 이어진다.
3. PDP에서 AI가 리뷰 요약, 사이즈 주의사항, 스타일링 팁, 가격 최적화까지 같이 제공한다.
4. 위시리스트가 정적 보관함이 아니라 `AI 옷장/스타일리스트`의 입력 데이터가 된다.

## 2. 목표와 비목표

### 2-1. 제품 목표
1. 사용자가 첫 방문 3분 안에 추천 상품 1개 이상을 저장하거나 상세를 열어보게 만든다.
2. 대화형 추천을 통해 일반 검색 대비 상품 탐색 시간을 줄인다.
3. 스타일 프로필과 행동 데이터를 활용해 재방문 시 추천 정확도를 개선한다.
4. 가격 추적, 재입고 알림, 위시 기반 리텐션 루프를 만든다.

### 2-2. 비즈니스 목표
1. 대화 시작률을 높여 신규 방문자의 탐색 이탈을 줄인다.
2. 추천 기반 구매 전환율을 확보한다.
3. 상/하의/신발/액세서리 연관 추천으로 AOV를 높인다.
4. 위시, 알림, 프로필 데이터를 통해 CRM 효율을 높인다.

### 2-3. 비목표
1. 초기 출시에서 소셜 커뮤니티를 메인 가치로 두지 않는다.
2. 초기 출시에서 AR 가상 피팅을 필수로 포함하지 않는다.
3. 초기 출시에서 모든 CS 업무 자동화를 목표로 하지 않는다.

## 3. 타깃 사용자

### 3-1. 핵심 타깃
1. 20~30대 직장인: 출근룩, 데이트룩, 하객룩 등 상황별 추천이 필요한 사용자
2. 스타일 비확신 사용자: 본인 취향은 있지만 언어화가 어려운 사용자
3. 효율 추구형 사용자: 긴 탐색보다 빠른 추천과 구매 의사결정을 선호하는 사용자
4. 패션 관심 중상 사용자: 위시 저장, 가격 추적, 코디 추천 니즈가 있는 사용자

### 3-2. 대표 페르소나

#### 페르소나 A: 바쁜 직장인 민지
- 니즈: 출근/약속 상황에 맞는 룩을 빠르게 추천받고 싶다.
- 불편: 검색어를 뭘 넣어야 할지 모르고, 리뷰를 일일이 읽기 귀찮다.
- 기대: "오늘 회식 있는 날 입을 블라우스 추천해줘" 한 문장으로 끝내고 싶다.

#### 페르소나 B: 스타일 탐색형 준호
- 니즈: 비슷한 분위기의 상품을 비교하고, 가격이 떨어질 때 사고 싶다.
- 불편: 앱을 여러 개 돌며 가격/혜택을 비교해야 한다.
- 기대: 가격 인사이트와 혜택 계산까지 한 번에 보고 싶다.

#### 페르소나 C: 옷장 활용형 서연
- 니즈: 이미 가진 옷과 어울리는 아이템을 사고 싶다.
- 불편: 단품은 예쁜데 기존 옷과 조합될지 모르겠다.
- 기대: 내 옷장 기반 코디 추천과 소재/세탁 설명이 필요하다.

### 3-3. JTBD
1. "약속 전에 빨리 옷을 사야 할 때, 상황에 맞는 추천을 받아 시간을 줄이고 싶다."
2. "예쁜 상품을 봤을 때, 나한테 어울리는지와 기존 옷과의 조합을 확인하고 싶다."
3. "관심 상품이 생겼을 때, 더 좋은 가격이나 구매 시점을 놓치고 싶지 않다."

## 4. 성공 지표 (KPIs)

### 4-1. 핵심 KPI
1. 대화 시작률: 앱 실행 후 첫 세션 내 메시지 1회 이상 전송 비율
2. 추천 상세 진입률: 추천 카드 노출 대비 PDP 진입 비율
3. 추천 기반 구매 전환율: 대화 세션에서 추천된 상품이 구매로 이어진 비율
4. 저장 전환율: 추천 상품 저장/위시 추가 비율
5. 가격 추적 설정률: PDP 진입 사용자 중 목표가/재입고 알림 설정 비율
6. D7, D30 리텐션
7. 평균 주문 금액(AOV)

### 4-2. 보조 KPI
1. 프로필 완성률
2. AI 옷장 아이템 등록률
3. 위시리스트 재방문율
4. 푸시 알림 오픈율
5. 대화당 평균 메시지 수
6. AI 추천 만족도

---

## 5. 핵심 제품 원칙

1. 앱의 첫 화면은 `대화 가능한 탐색`이어야 한다.
2. 모든 추천에는 최소 1개의 `이유 설명`이 따라야 한다.
3. 추천 결과는 대화가 끊기지 않도록 `후속 액션`을 제공해야 한다.
4. 상품 정보는 구매 결정을 돕는 방향으로 요약되어야 한다.
5. AI는 절대 존재하지 않는 상품이나 재고 없는 상품을 확정형으로 추천하면 안 된다.

```mermaid
flowchart TD
    Launch[앱실행] --> Onboarding[온보딩/로그인]
```

## 8. 상세 기능 요구사항

## 8-1. 계정 및 인증

### 목표
가입 허들을 낮추고, 첫 세션에서 대화 시작까지 빠르게 연결한다.

### 기능
1. 이메일 로그인/회원가입
2. 소셜 로그인: 카카오, 애플, 구글
3. 비로그인 상태 탐색 허용 여부 결정
4. 로그인 후 이전 비로그인 위시/최근 대화 병합
5. 약관/개인정보 동의

### 세부 요구사항
1. 앱 최초 실행 시 온보딩 후 로그인 유도
2. 비회원도 대화 1회 체험 가능하도록 선택지 검토
3. 회원가입 완료 후 즉시 스타일 프로필 입력 화면으로 이동
4. 로그인 실패 시 원인별 메시지 제공
5. 애플 로그인은 iOS 필수 지원

### 예외/상태
1. 탈퇴 회원 재가입 처리 정책 필요
2. 소셜 계정 중복 가입 병합 정책 필요

### 분석 이벤트
`signup_started`, `signup_completed`, `login_completed`, `guest_trial_started`

## 8-2. 온보딩 및 스타일 프로필

### 목표
AI 추천 정확도를 높이는 최소한의 개인화 데이터를 수집한다.

### 입력 항목
1. 성별
2. 나이대
3. 키
4. 체형 정보 또는 체형 고민
5. 상의/하의 선호 핏
6. 평소 사이즈
7. 선호 브랜드
8. 선호 스타일
9. 기피 색상/기피 스타일
10. 개인 컬러
11. 예산 범위
12. 예산 유연성

### 세부 요구사항
1. 프로필 입력은 3~5 스텝으로 분리해 부담을 줄인다.
2. 선택형 칩 중심 UI로 빠르게 입력 가능해야 한다.
3. 필수/선택 항목을 구분한다.
4. 프로필은 마이페이지에서 언제든 수정 가능해야 한다.
5. AI 추천 호출 시 프로필 정보가 컨텍스트로 반영되어야 한다.

### MVP 범위
1. 성별
2. 키
3. 나이대
4. 선호 브랜드
5. 선호 스타일
6. 예산 범위

`profile_started`, `profile_step_completed`, `profile_completed`, `profile_updated`

## 8-4. 대화형 상품 탐색

### 목표
검색창 대신 자연어 기반 쇼핑을 제공한다.

### 입력 형태
1. 자유 텍스트 입력
2. 빠른 질문 칩
3. 음성 입력
4. 이미지 업로드 (Phase 2)

### 세부 요구사항
1. 사용자는 TPO, 색상, 예산, 핏, 브랜드, 계절감 등을 문장으로 입력할 수 있어야 한다.
2. AI는 입력 의도를 분류해야 한다.
3. AI 응답은 `설명 텍스트 + 상품 카드 + 후속 질문` 구조를 기본으로 한다.
4. 채팅 응답은 스트리밍 방식으로 보여준다.
5. 추천이 애매할 경우 추가 질문으로 명확도를 높인다.
6. 사용자가 "더 저렴한 것", "비슷한 스타일", "다른 브랜드" 같은 후속 질문을 쉽게 이어갈 수 있어야 한다.
7. 추천 결과가 없을 때는 인기 상품 또는 유사 카테고리 대안을 제시한다.

### AI 응답 카드 필수 요소
1. 상품 이미지
2. 브랜드명

## 8-6. 상품 상세(PDP)

### 필수 정보 영역

2. 브랜드명/상품명/기본 가격

8. 소재/세탁/주의사항
9. 유사 상품
10. 가격 비교/혜택 계산
11. 저장/공유
12. 장바구니/바로 구매

### 세부 요구사항
1. 선택된 색상에 따라 이미지와 재고가 변경되어야 한다.
2. 품절 컬러는 선택 가능하되 구매 버튼은 비활성화하고 재입고 알림을 제공한다.

5. 가격 탭에서는 플랫폼별 총 결제 예상가와 혜택을 비교한다.
6. 저장 버튼은 위시리스트와 즉시 연동된다.

### 확장 기능
1. "이 상품과 잘 어울리는 아이템 추천"

### MVP 범위
1. 위시 상품 기반 추천
2. 간단한 내 아이템 수동 등록
3. 텍스트 기반 코디 추천

### Phase 2 범위
1. 이미지 업로드 기반 내 옷 등록
2. 소재/세탁/유사 상품 분석
3. 가상 착장 이미지 또는 조합 카드

## 11. 비기능 요구사항

### 11-1. 성능

3. 추천 카드 렌더 완료는 2.5초 이내 목표

1. AI 실패 시 규칙 기반 또는 인기 상품 fallback 제공
2. 결제 실패 시 재시도 및 복구 플로우 필요
3. 네트워크 불안정 시 임시 저장/재전송 고려

1. 결제 관련 정보는 PG 위임 처리
2. 프로필과 체형 정보는 암호화 저장
3. 접근 권한 및 로그 감사를 지원

### 11-4. 데이터 무결성
1. 추천 가능한 상품만 응답해야 한다.
2. 가격, 재고, 옵션은 실시간 또는 준실시간 동기화 필요
3. 삭제/품절 상품 추천 방지 로직 필요

### 11-5. 접근성
1. 핵심 버튼 터치 영역 최소 44px

3. 개인화 프로필을 바탕으로 기본 추천이 가능하다.

### 포함 기능
1. 로그인/회원가입
2. 온보딩 및 스타일 프로필
3. 홈
4. 대화 검색
5. 추천 결과 카드
6. PDP 기본형
7. 위시리스트
8. 장바구니/결제
9. 주문 내역
10. 기본 어드민

## 15. 리스크 및 대응

### 리스크
1. AI 환각으로 존재하지 않는 상품 추천 가능성
2. 상품 데이터 정합성 부족
3. 추천 품질 편차
4. 가격 비교 데이터 수집 난이도
5. 결제/주문 운영 복잡도

### 대응
1. 상품 ID 기반 구조화 응답 강제
2. fallback 추천 체계 구축
3. 추천 로그 및 품질 평가 대시보드 운영
4. 초기에는 내부 가격/프로모션 기준으로 시작
5. CS 자동화는 단계적으로 도입

## 18. 최종 정리

이 앱의 본질은 `대화형 탐색`이다.  
따라서 MVP에서는 검색을 대체하는 대화 경험과 실제 구매 루프를 먼저 완성하고, 이후 `AI 옷장`, `가격 추적`, `멀티모달`, `리텐션 알림`을 붙여 `AI 스타일리스트 앱`으로 진화시키는 전략이 가장 적합하다.
